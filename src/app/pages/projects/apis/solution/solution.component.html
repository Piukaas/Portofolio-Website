Ball.com

For Ball.com, a fake retailer going global, the project involved designing a system where customers can place orders from a pre-determined set of products. Orders are picked from a large warehouse, packaged, and sent using a logistics company selected by lowest price. Trusted third-party suppliers can add products. Customers can track order status and pay using forward- or after-payment. A service department answers customer queries. The system was designed using microservices architecture, event-driven communication, and eventual consistency to ensure scalability, maintainability, and efficiency.
